# SELinuxの再来

### SELinuxの仕組み

SELinux（Security-Enhanced Linux）は、Linuxカーネルに強力なアクセス制御機能を追加するセキュリティーモジュールです。SELinuxは、従来のUNIX権限モデルを補完するために開発され、細かいアクセス制御ポリシーを定義することができます。SELinuxのアクセス制御は、ユーザー、プロセス、ファイル、リソースなどの属性に基づいて行われます。

### コンテキスト

SELinuxでは、各オブジェクト（ファイルやプロセスなど）にはコンテキストと呼ばれるラベルが付与されます。コンテキストは、通常、ユーザー名、ロール名、タイプ（またはドメイン）名、そしてオプションでレベル（または範囲）名から構成されます。例えば、以下のようなコンテキストが考えられます：

```text
user_u:role_r:type_t:s0
```

### ポリシー

SELinuxのポリシーは、どのユーザーがどのリソースに対してどのような操作を行えるかを定義します。これにより、アクセス制御のルールを細かく定義することができます。例えば、次のようなポリシーを定義できます：

- あるタイプのプロセスが別のタイプのファイルに対して読み取り操作を行える
- あるロールのユーザーが特定のタイプのリソースに対して書き込み操作を行える

### 強制アクセス制御（MAC）

SELinuxは強制アクセス制御（Mandatory Access Control, MAC）を実現します。これは、システム管理者が設定したポリシーに従ってアクセス制御が行われるため、ユーザーやアプリケーションが自由に権限を変更することができません。これにより、セキュリティ上のリスクを大幅に軽減することができます。

### SELinuxによるroot特権の制限

SELinuxは、root特権を持つプロセスの危険性を軽減する目的でも利用されます。伝統的なLinuxの権限モデルでは、root権限を持つプロセスは非常に広い操作が可能であり、攻撃者がこれを悪用することでシステム全体に影響を与える可能性がありました。SELinuxを導入することで、root特権を持つプロセスにもポリシーによる制限をかけることができ、攻撃者がシステム全体に影響を与えるリスクを減らすことができます。

### 権限管理の改善とSELinux

SELinuxは、伝統的な権限管理だけでは表現しにくいアクセス制御を追加するために利用されます。従来のUNIX権限では、ファイルの所有者、グループ、その他のユーザーという粒度が中心でした。しかし、SELinuxを使用することで、プロセスやファイルに付いたタイプに基づき、サービスごとに必要なアクセスだけを許可できます。これにより、システムのセキュリティが向上し、攻撃された場合の被害を限定することができます。

### SELinuxの活用

SELinuxは、特に高いセキュリティが求められる環境や、複数のユーザーやアプリケーションが共存するシステムで効果を発揮します。以下は、SELinuxを活用する具体的なシナリオです：

- ウェブサーバー：ウェブサーバーはインターネットからの攻撃に直接さらされるため、セキュリティが重要です。SELinuxを用いることで、ウェブサーバープロセスが不必要なファイルやリソースにアクセスすることを防ぐことができます。
- データベースサーバー：データベースサーバーは機密情報を保持することが多いため、アクセス制御が重要です。SELinuxを用いることで、データベースプロセスが不正なアクセスを受けた場合でも、被害を最小限に抑えることができます。
- クラウド環境：クラウド環境では、複数のユーザーやアプリケーションが共存するため、細かいアクセス制御が求められます。SELinuxを用いることで、各ユーザー間やアプリケーション間でのリソースのアクセス制御を実現することができます。これにより、あるユーザーやアプリケーションが他のユーザーやアプリケーションのリソースに不正アクセスすることを防ぐことができます。
- エンドポイント保護：エンドポイント（ワークステーションやラップトップなど）は、セキュリティ上の脆弱性がある場合があります。SELinuxを用いることで、マルウェアや他のセキュリティ上の脅威からエンドポイントを保護することができます。
