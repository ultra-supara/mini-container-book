# Linuxの権限管理とセキュリティまわり

## Linuxカーネルの提供する機能

### 入出力

ファイルの読み書きやネットワークへのアクセス、周辺機器へのアクセスなどがあります.

### プロセスやユーザなどの管理

プロセスを終了させたり、ユーザを追加したり権限を付与したりなどがあります.

### プロセスのリソース制限

Linuxカーネルは、プロセスに対してCPU使用量、メモリ使用量、オープンできるファイル数などのリソース制限を設けることができます。これにより、システム全体のリソースを適切に管理・配分できます。

### ユーザーごとの権限の割り当て

どのような操作が許されるかはユーザーやグループごとに決まっています.

### プロセスとユーザーの権限関係

プロセスは、実行中のユーザーによって権限が決まります。ただし、プロセスはそれぞれ異なる権限を持つこともあります。これにより、特定のプロセスが特権操作を行うことができるように設定できます。

### カーネルの権限チェック

Linuxカーネルの機能はシステムコールを通して利用します. あるプロセスが権限ある操作をしようとしたとき, それが許されるかどうかはシステムコールを呼んだときにカーネルがチェックします.

### ファイルシステムへのアクセス権限

ファイルシステム上のファイルやディレクトリは、設定されたパーミッションによってアクセス権限が制御されます。これには、読み込み、書き込み、実行の権限があります。

### 読み、書き、実行の適用

パーミッションには、ファイルやディレクトリに対する読み込み権限、書き込み権限、実行権限があります。これらの権限は、所有ユーザー、所有グループ、その他のユーザーに対してそれぞれ適用されます。

### 所有ユーザー、所有グループ

ファイルやディレクトリは所有ユーザー、所有グループ、その他のユーザーに対してそれぞれ異なるパーミッションを設定できます。例えば、所有ユーザーには読み書き実行のすべての権限を付与し、所有グループには読みと実行のみを許可し、その他のユーザーには読み込みのみを許可する、といった設定が可能です。

### その他の操作に必要な権限

ファイル操作以外にも、特権ユーザーにしか許されない操作があり、たとえばシャットダウンやuidの異なるプロセスを終了させるなどです. それにroot特権が必要かどうかはシステムコールによります .

### プロセスのユーザーの振る舞い

プロセスは、ユーザーID（UID）とグループID（GID）を持ち、これらに基づいて権限チェックが行われます。また、プロセスは、実行中の操作やリソースアクセスに対して権限チェックが適用されるため、適切な権限が必要です。

### プロセスに割り当てられる様々なユーザーID

1. **euid(Effective User ID)**

多くの権限チェックで参照されるユーザーIDです。ファイルアクセスでは厳密にはfsuidも関係し、`chroot`や`mount`などの特権操作ではcapabilityも関係しますが、まずはeuidが重要だと理解しておけば十分です。

2. **suid(Saved User ID)**

一時的にeuidを変更するとき、元のユーザーに戻ることができるようにするためのものです.

3. **ruid(Real User ID)**

プロセスを所有しているユーザーのIDでシグナルの送信に関する権限チェックに用いられます .

### ユーザーIDの変更

**`setuid`**、**`seteuid`**、**`setreuid`** などの関数を使用して、プロセスのユーザーIDを変更することができます。ただし、rootユーザーのみ任意のユーザーIDに変更できます。一般ユーザーは、**`ruid`**か**`suid`**のユーザーIDにのみ変更できます。

### 子プロセスを作るとき

あるプロセスが別のプロセスを作ると、その子プロセスのそれぞれのユーザーは親プロセスと同じになります. 子や親が`uid`を後から変更しても、その変更は独立したプロセスであり親や子に伝播しないです.

### 子プロセスの権限とユーザーID

親プロセスが子プロセスを生成する際、通常は子プロセスは親プロセスと同じユーザーIDと権限を持ちます。ただし、子プロセスは、親プロセスから独立してユーザーIDや権限を変更できます。例えば、セットUIDビットが設定された実行ファイルを実行すると、子プロセスは実行ファイルの所有ユーザーと同じ権限を持つことになります。これにより、特定の操作が許可されるようになります。

### 操作の実例

1. **chmod**

ファイルやディレクトリのパーミッションを変更するためのコマンドです。例えば、所有者に読み書き権限を与え、グループとその他のユーザーには読み取り権限のみを与える場合、次のように実行します。

```bash
chmod 644 example.txt
```

1. **chown**

ファイルやディレクトリの所有者を変更するためのコマンドです。例えば、所有者を'user1'、所有グループを'group1'に変更する場合、次のように実行します。

```bash
chown user1:group1 example.txt
```

1. **su**

現在のユーザーから別のユーザーに切り替えるためのコマンドです。例えば、rootユーザーに切り替える場合、次のように実行します。

```bash
su - root
```

1. **sudo**

特定のコマンドを特権ユーザーとして実行するためのコマンドです。例えば、システムの更新を行う場合、次のように実行します。

```bash
sudo apt update
```

これらのコマンドを用いることで、Linux上で権限管理やセキュリティを制御することが可能です。ただし、適切な権限設定がなされていない場合、システムのセキュリティが低下し、悪意のある攻撃者による悪用が可能になるため注意が必要です。

### ユーザーとグループの作成と削除

1. **useradd**

新しいユーザーを作成するためのコマンドです。例えば、新しいユーザー'user2'を作成する場合、次のように実行します。

```bash
sudo useradd user2
```

1. **userdel**

既存のユーザーを削除するためのコマンドです。例えば、ユーザー'user2'を削除する場合、次のように実行します。

```bash
sudo userdel user2
```

1. **groupadd**

新しいグループを作成するためのコマンドです。例えば、新しいグループ'group2'を作成する場合、次のように実行します。

```bash
sudo groupadd group2
```

1. **groupdel**

既存のグループを削除するためのコマンドです。例えば、グループ'group2'を削除する場合、次のように実行します。

```bash
sudo groupdel group2
```

### ユーザーとグループの関連付け

1. **usermod**

ユーザーの属性を変更するためのコマンドです。例えば、ユーザー'user1'をグループ'group1'に追加する場合、次のように実行します。

```bash
sudo usermod -aG group1 user1
```

1. **groups**

指定したユーザーが所属するグループを表示するためのコマンドです。例えば、ユーザー'user1'が所属するグループを表示する場合、次のように実行します。

```bash
groups user1
```

### ファイルとディレクトリのアクセス制御

1. **umask**

デフォルトのファイルやディレクトリのパーミッションを設定するためのコマンドです。例えば、新規作成されるファイルに対して、所有者には読み書き権限を与え、グループとその他のユーザーには読み取り権限のみを与える場合、次のように実行します。

```bash
umask 022
```

これらのコマンドを使用して、Linux上で権限管理やセキュリティを効果的に制御することができます。ただし、権限設定を適切に行わないと、システムのセキュリティが低下し、悪意のある攻撃者による悪用が可能になるため注意が必要です。

### スティッキービットとセットUID

1. **スティッキービット**

スティッキービットは、ディレクトリに特別な制限を設定するために使用されます。これにより、ディレクトリ内のファイルは、所有者だけが削除やrenameができるようになります。スティッキービットを設定するには、次のようなchmodコマンドを実行します。

```bash
chmod +t directory_name
```

1. **セットUID**

セットUIDは、特定のファイルに対して、実行時にファイル所有者の権限で実行されるように設定するものです。これにより、通常は権限がないユーザーでも、そのファイルを実行することができるようになります。セットUIDを設定するには、次のようなchmodコマンドを実行します。

```bash
chmod u+s file_name
```

ただし、セットUIDを適切に設定しないと、セキュリティリスクが高まるため、注意が必要です。

### ACL（Access Control List）によるアクセス制御

1. **setfacl**

Linuxでは、ACL（Access Control List）を使用して、ファイルやディレクトリに対してより詳細なアクセス制御を行うことができます。setfaclコマンドを使用して、特定のユーザーまたはグループに対してファイルやディレクトリのアクセス権を設定できます。

例えば、ユーザー'`user1`'に対してファイル'`file1.txt`'の読み取り権限を設定する場合、次のように実行します。

```bash
setfacl -m u:user1:r file1.txt
```

1. **getfacl**

getfaclコマンドを使用して、ファイルやディレクトリのACL設定を表示できます。例えば、ファイル'file1.txt'のACL設定を表示する場合、次のように実行します。

```bash
getfacl file1.txt
```

これらの概念やコマンドを理解し、適切に適用することで、Linuxシステムにおける権限管理やセキュリティ対策が効果的に行えます。ただし、権限の設定を誤るとシステムのセキュリティが低下し、悪意のある攻撃者による悪用が可能になるため、注意が必要です。
