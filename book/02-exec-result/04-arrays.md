# 配列について

配列を宣言すると，メモリ上にその型の領域の連続が確保されます．Cでは，配列式は多くの場面で先頭要素へのポインタに変換されますが，配列そのものがポインタ変数になるわけではありません． たとえば，`int`型の配列`int arr[3]`を宣言した場合を考えてみます．ここでは，Linux上の多くの環境と同じように`int`型のサイズが4バイトで，`arr[0]`の先頭アドレスが`0x1234`であるとします．

```text
+---------------+---------------+----------------+
| Memory Address| Array Content | Pointer Notation|
+---------------+---------------+----------------+
|    0x1234     |               |     arr        |
|    0x1235     |    arr[0]     |                |
|    0x1236     |               |                |
|    0x1237     |               |                |
+---------------+---------------+----------------+
|    0x1238     |               |     arr + 1    |
|    0x1239     |    arr[1]     |                |
|    0x123a     |               |                |
|    0x123b     |               |                |
+---------------+---------------+----------------+
|    0x123c     |               |     arr + 2    |
|    0x123d     |    arr[2]     |                |
|    0x123e     |               |                |
|    0x123f     |               |                |
+---------------+---------------+----------------+
```

つまり，メモリ上のアドレス`0x1234`から`0x123f`にかけてのメモリが12バイト確保され，`arr[0]`の値は`0x1234`から4バイトにわたって記録されます．ポインタの操作では，1バイトごとではなくそのポインタの指す値の型のサイズごと，つまりこの場合は`int`型で4バイトごとに進むので，`arr + 1`が指すアドレスは`0x1234 + 0x4 = 0x1238`となります．このように、配列とポインタは密接に関連しており、配列の要素にアクセスする際にポインタ演算が頻繁に使用されます.
