# IPv6への対応

IPv6環境をサポートする場合は、対応するIPv6アドレスとルート設定が必要です。IPv6ではNATが一般的に推奨されていないため、可能な限りグローバルなIPv6アドレスを直接割り当てて、NATを避けることが望ましいです。

グローバルなIPv6アドレスを割り当てるために、ルーターやISPからのアドレス情報を取得します。DHCPv6やSLAAC（Stateless Address Autoconfiguration）が用いられることが一般的です。

ルーターでIPv6ルーティングを有効にします。以下のコマンドを実行して、IPv6フォワーディングを有効にしてください。

```bash
$ sysctl -w net.ipv6.conf.all.forwarding=1
```

永続的に設定する場合は、**`/etc/sysctl.conf`** または **`/etc/sysctl.d/`** にある適切なファイルに以下の行を追加してください。

```bash
net.ipv6.conf.all.forwarding=1
```

ルーターのインターフェイスにIPv6アドレスを割り当てます。例えば、プレフィックスが **`2001:db8:1::/64`** の場合、インターフェイスに以下のようにアドレスを割り当てます（インターフェイス名を適切なものに置き換えてください）。`2001:db8::/32`はドキュメント用の例示プレフィックスなので、実際の通信にはISPやルーターから割り当てられたプレフィックスを使ってください。

```bash
$ ip -6 addr add 2001:db8:1::1/64 dev ens4
```

ルーターで、内部ネットワークに対するルート広告を有効にします。これにより、内部ネットワークのデバイスが自動的にグローバルなIPv6アドレスを割り当てられ、ルーティングが行われます。**`radvd`** や **`dnsmasq`** のようなルート広告デーモンを使用して設定します。**`/etc/radvd.conf`** などの設定ファイルに以下のような設定を追加してください。

```text
interface ens4 {
    AdvSendAdvert on;
    prefix 2001:db8:1::/64 {
        AdvOnLink on;
        AdvAutonomous on;
    };
};
```

その後、ルート広告デーモンを起動・再起動します。

```bash
$ systemctl restart radvd
```

これで、内部ネットワークのデバイスは自動的にグローバルなIPv6アドレスを割り当てられ、インターネットにアクセスできるようになります。また、IPv6を使用することで、端末間の直接通信が容易になり、P2PアプリケーションやVoIPなどの通信品質も向上します。
