# ここでは, 名前空間について説明します.

### 概要

Linuxでは，いくつかの資源を各プロセス共通で利用しています． たとえばネットワークやホスト名，プロセス間通信，ユーザID，時刻などがあります．それらを**プロセスごとに別々に見せるためのしくみ**が名前空間です．何が分離されるかは名前空間の種類によって異なり，たとえばNetwork名前空間ではネットワークインターフェイスやルーティングテーブルの見え方が分かれます． これとは別に，ハードウェア資源の使用量を制限するための`cgroups`というしくみもあります．

### 応用例

まず，Dockerなどのコンテナに利用されています．コンテナの分離の一部はLinuxの名前空間で実現されています．たとえば，Network名前空間を利用することで，Dockerコンテナ内部のアプリケーションからはホスト側のネットワークインターフェイスが直接見えにくくなります．コンテナ内でroot権限を持っていても，ホストのNetwork名前空間にあるインターフェイスをそのまま操作できるわけではありません．Dockerで使われている`runc`も名前空間を利用しています.そのほかにも，ブラウザなどにも安全に実行するために名前空間を利用するものがあります．

## 種類

名前空間には様々な種類があります．種類によって分離するものが違います．

| **種類** | **分離するもの** |
| --- | --- |
| IPC | System V IPC, POSIX message queues |
| Network | Network devices, stacks, ports, etc. |
| Mount | Mount points |
| PID | Process IDs |
| Time | 一部の時計のオフセット |
| User | User and group IDs |
| UTS | Hostname and NIS domain name |

## 仕様

### プロセスごと

名前空間はそれぞれのプロセスごと, リソースごとに割り当てられています．つまり，プロセスごとに違う名前空間を利用して，プロセスごとにネットワークなどを分離することができます．複数のプロセスを1つの名前空間に所属させ，共有することもできます．

### 種類ごと

すべてのプロセスはそれぞれすべての種類の名前空間を持ちます．種類ごと，プロセスごとに名前空間を設定できます．

### 起動時

起動直後，すべてのプロセスは同じデフォルトの名前空間に属しています．

### 削除

基本的に，その名前空間に属するプロセスがなくなると，その名前空間は削除されます．プロセスが無くなっても削除されない，いくつかの例外があります．

### root特権の必要性

多くの名前空間の作成や操作には，現在のUser名前空間での`CAP_SYS_ADMIN`などの権限が必要になります．一方，User名前空間は一般ユーザでも作成できる設定の環境があります．ただし，ディストリビューションやカーネル設定によっては，一般ユーザによるUser名前空間の作成が無効化されていることもあります．

### システムコールでの操作

以下のシステムコールを用いて名前空間を操作できます．

- clone: 子プロセスを作成し、新しい名前空間を割り当てる。
- unshare: 現在のプロセスから名前空間を分離する。
- setns: 現在のプロセスを別の名前空間に移動する。
- ioctl: 特定のデバイスや名前空間の設定を操作する。

### プロトタイプ

```c
int clone(int (*fn)(void *), void *stack, int flags, void *arg, ...
                 /* pid_t *parent_tid, void *tls, pid_t *child_tid */ );
```

第3引数の`flags`を設定することで，新しく名前空間を作成し，その名前空間内でプロセスを開始できます．それぞれの種類の名前空間を作成するには，以下の定数を使用します．複数を指定するには，ビットごとにORします．

| 種類 | 定数 |
| --- | --- |
| IPC | `CLONE_NEWIPC` |
| Network | `CLONE_NEWNET` |
| Mount | `CLONE_NEWNS` |
| PID | `CLONE_NEWPID` |
| Time | `CLONE_NEWTIME` |
| User | `CLONE_NEWUSER` |
| UTS | `CLONE_NEWUTS` |

これらの定数を**`flags`**に設定して**`clone`**を呼び出すことで、指定した種類の名前空間を新しく作成し、その名前空間内でプロセスが開始されます。
