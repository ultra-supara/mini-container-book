# シグナル

### 概要

シグナルは、プロセス間で定型的なメッセージを送信する機能です。これにより、プロセスの状態を制御したり変更したりすることができます。シグナルを利用して、あるプロセスが別のプロセスに対して終了を要求したり、一時停止や再開を指示したりできます。

### 種類と例

シグナルには多くの種類があり、それぞれが異なる目的を持っています。以下は一部のシグナルの例です。

- `SIGHUP`: プロセスがハングアップを検出したことを通知する。
- `SIGINT`: プロセスに割り込みを要求し、通常はプロセスを終了させる。
- `SIGQUIT`: プロセスに終了を要求し、コアダンプを生成する。
- `SIGKILL`: プロセスを強制終了させる。
- `SIGSTOP`: プロセスを一時停止させる。
- `SIGCONT`: 一時停止したプロセスを再開させる。

### シグナルの送信方法

シグナルは主に`kill`コマンドを使って送信されますが、他にも`kill`システムコールやキーボードショートカット（例: Ctrl+CでSIGINT）などで送信することができます。

### シグナルのハンドリング

プロセスは、受け取ったシグナルに対して特定の動作を実行することができます。この動作はプログラム内で定義されたシグナルハンドラによって行われます。シグナルハンドラが設定されていない場合、シグナルのデフォルト動作が実行されます。

### デフォルト動作

シグナルのデフォルト動作は、シグナルの種類によって異なります。一部のシグナルはプロセスを終了させ、他のシグナルはプロセスを一時停止させたり、無視したりします。

### 権限と送信制限

シグナルを送信できるのは、通常、シグナルの送信元のプロセスと送信先のプロセスが同じユーザーによって所有されている場合です。ただし、rootユーザーはすべてのプロセスに対してシグナルを送信できます。また、`CAP_KILL`権限を持つプロセスは、他のプロセス所有者に対してもシグナルを送信できます。

### 練習

`SIGKILL`シグナルを受信したプロセスは終了します．これを試してみます．まず，`sleep`コマンドをバックグラウンドで実行します．

```bash
$ sleep 3600 &
```

すると，実行が開始されたことを示すメッセージが表示されます．

```text
[1] 3846
```

`ps`コマンドで，`sleep`コマンドのプロセスが実行されていることを確認しましょう．

```bash
$ ps
```

すると，以下のように`sleep`コマンドのプロセスが実行されていることが確認できます．

```bash
    PID TTY          TIME CMD
   2235 pts/0    00:00:00 bash
   3846 pts/0    00:00:00 sleep
   3847 pts/0    00:00:00 ps
```

この場合，実行中の`sleep`コマンドのプロセスのプロセスIDは`3846`です． このプロセスに`SIGKILL`シグナルを送信してみましょう．`kill`コマンドは指定したプロセスにシグナルを送信するコマンドです．以下のように`-s SIGKILL`オプションを指定すると，`SIGKILL`シグナルを送信することができます．その後にプロセスIDを入力し，実行します． プロセスIDは環境によって違うため，先ほど`ps`コマンドで表示されたものを使用してください．

```bash
$ kill -s SIGKILL 3846
```

その後，再び`ps`コマンドでプロセス一覧を表示します．すると，プロセス一覧から先ほどのプロセスIDが`3846`のプロセスがなくなっていることが確認できます．さらに，最後の行にはプロセスが`SIGKILL`シグナルによって終了した旨のメッセージが表示されます．

```bash
    PID TTY          TIME CMD
   2235 pts/0    00:00:00 bash
   3851 pts/0    00:00:00 ps
[1]+  Killed                  sleep 3600
```
