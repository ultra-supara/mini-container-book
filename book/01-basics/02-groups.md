# グループ

### 概要

Linuxでは，複数のユーザが属するグループを扱うことができます．先述したユーザに加えて，操作するユーザの属するグループの権限によって，ファイルへのアクセスを許可することができます．1つのユーザは複数のグループに属することができます．

### グループID

ユーザの場合と同じように，グループも0以上の整数のIDを割り当てられています．省略してGIDと呼ばれます．ユーザIDのときと同じように，プログラムはグループをこのGIDによって扱うことが多いです．

### 練習

```bash
$ id
```

結果は，概ね以下のような形式になっていると思います．

```bash
uid=1000(para) gid=1000(para) groups=1000(para),27(sudo)
```

`gid=`から始まるところは，そのユーザの主グループを示します．また，`groups=`以降には`,`で区切られた所属グループの一覧が表示されています．この例では，`para`というグループのグループIDは`1000`で，`sudo`というグループのグループIDは`27`です．DebianやUbuntu系の環境では，`sudo`グループに属しているユーザが`sudo`コマンドを利用できます．Red Hat系の環境では，代わりに`wheel`グループが使われることもあります．`groups`というコマンドを用いても，所属しているグループを調べることができます．`$ groups` コマンドは、現在のユーザーが所属するグループを表示します. このコマンドを実行して、自分のグループ情報を確認しましょう.

```bash
$ groups
```

このように表示されます．さきほどと同じグループが表示されていることが分かります．

```bash
para sudo
```

### グループの作成・削除

新しいグループを作成するには、`groupadd` コマンドを使用します。以下の例では、新しいグループ「mygroup」を作成しています。

```bash
$ sudo groupadd mygroup
```

グループを削除するには、`groupdel` コマンドを使用します。以下の例では、先ほど作成した「mygroup」グループを削除しています。

```bash
$ sudo groupdel mygroup
```

### ユーザをグループに追加・削除

ユーザをグループに追加するには、`usermod` コマンドを使用します。以下の例では、ユーザ「johndoe」を「mygroup」グループに追加しています。

```bash
$ sudo usermod -a -G mygroup johndoe
```

ユーザをグループから削除するには、`gpasswd` コマンドを使用します。以下の例では、ユーザ「johndoe」を「mygroup」グループから削除しています。

```bash
$ sudo gpasswd -d johndoe mygroup
```

### グループに関する権限管理

ファイルやディレクトリの権限設定に関して、グループに関するアクセス権限（読み取り、書き込み、実行）を説明します。以下の例では、ファイル「sample.txt」のグループ権限を読み取りと実行に設定しています。

```bash
$ chmod g=rx sample.txt
```

### /etc/group ファイル

グループ情報は `/etc/group` ファイルに格納されています。このファイルを見ることで、システム上のすべてのグループとそのメンバーを確認できます。

```bash
$ cat /etc/group
```
