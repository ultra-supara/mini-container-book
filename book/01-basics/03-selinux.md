# SELinux

### ユーザとグループによる権限管理の問題点

これまで，ユーザとグループによる権限について学んできました．これらは有用ですが，充分ではない部分があります．それは**許可する操作を限定すること**です．OSの一般ユーザには許可されていない機能を利用するとき，ユーザとグループによる権限管理だと，rootユーザを利用するしかありませんでした．その他にも，ファイルへのアクセス制御でも，グループでは不十分な場合がありました．

### rootユーザの危険性

rootユーザは非常に強い権限を持つため，何らかのセキュリティ上の問題によってrootユーザを悪意ある人に操作されてしまうと，システム全体に大きな影響が出ます．つまり，rootユーザは強力すぎるため，扱いにはたいへん気を遣うものです．

### 解決策としてのSELinux

以上のような問題点を緩和するため，伝統的なユーザとグループによる権限管理に加えて，SELinuxという機能が用意されました．SELinuxは，プロセスやファイルなどにラベルを付け，ポリシーに従って「どの種類のプロセスが，どの種類のファイルや資源に，どの操作をしてよいか」を制御する強制アクセス制御（MAC）の仕組みです．rootユーザで動作するプロセスであっても，SELinuxポリシーによりアクセスを拒否できます．なお，特権を細かく分割して扱う仕組みはLinux capabilitiesであり，SELinuxとは別の機能です．

## SELinuxの設定

SELinuxは有用な機能ですが、設定が複雑であるため、正しく設定しなければなりません。以下に、SELinuxの基本的な設定方法を示します。

### SELinuxのモード

SELinuxは3つのモードで動作します。

1. **Enforcing**: SELinuxポリシーが有効で、違反が防止され、記録されます。
1. **Permissive**: SELinuxポリシーが有効ですが、違反は許可され、記録されます。これはデバッグやトラブルシューティングに役立ちます。
1. **Disabled**: SELinuxが無効で、ポリシーは適用されません。

モードを変更するには、`/etc/selinux/config`ファイルを編集します。以下のようにSELINUX変数を設定します。

```bash
SELINUX=enforcing
```

または

```bash
SELINUX=permissive
```

または

```bash
SELINUX=disabled
```

変更を反映させるためにシステムを再起動します。

### SELinuxポリシー

SELinuxポリシーは、システム上のオブジェクト（ファイル、ディレクトリ、ポートなど）とプロセス間のアクセス制御を定義します。主要なSELinuxポリシーは次のとおりです。

1. **Targeted**: 一部のデーモンやプロセスに対するアクセス制御が有効で、それ以外は無制限です。
1. **Strict**: システム全体に対するアクセス制御が有効で、すべてのプロセスが制限されます。

SELinuxポリシーを設定するには、`/etc/selinux/config`ファイルでSELINUXTYPE変数を設定します。

```bash
SELINUXTYPE=targeted
```

または

```bash
SELINUXTYPE=strict
```

### SELinuxの状態を確認する

SELinuxの状態やモードを確認するには、`sestatus`コマンドを使用します。

### ファイルのSELinuxコンテキストを変更する

ファイルやディレクトリのSELinuxコンテキストを変更するには、`chcon`コマンドを使用します。以下の例では、`/var/www/html/index.html`ファイルのコンテキストを`httpd_sys_content_t`に変更しています。

```bash
$ sudo chcon -t httpd_sys_content_t /var/www/html/index.html
```

### SELinuxの運用

SELinuxはLinuxシステムのセキュリティを強化する重要な機能ですが、適切に設定し運用することが重要です。以下に、SELinuxを適切に運用するためのベストプラクティスをいくつか示します。

1. **最小権限の原則**: 必要なアクセスだけを許可するポリシーを設定し、セキュリティリスクを最小限に抑えます。
1. **適切なドキュメント**: SELinuxの設定やポリシーを適切にドキュメント化し、管理者が理解しやすいようにしておくことが重要です。
1. **監査とログの管理**: SELinuxのアクセス制御違反や許可されたアクションについては、適切なログを生成するように設定し、ログを定期的に監査して不正なアクティビティを検出します。
1. **定期的なアップデート**: SELinuxのポリシーや設定を最新のセキュリティ要件に合わせて定期的に更新します。
1. **セキュリティ意識の向上**: 組織内でセキュリティ意識を高めるためのトレーニングや啓発活動を行い、SELinuxの重要性を理解し適切に運用することが重要です。

SELinuxは、システム全体のセキュリティを向上させるための強力なツールです。しかし、その複雑さから適切な設定や運用が求められます。上記のベストプラクティスを適用し、セキュリティリスクを最小限に抑えることが重要です。
