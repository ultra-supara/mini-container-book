# パイプ

### 概要

パイプは、プロセス間で通信を行うために使用される特殊なファイルです。2つのプロセスがそれぞれパイプを開き、片方のプロセスがパイプに書き込み、もう一方のプロセスがそのパイプから読み出すことで、通信が実現されます。パイプはファイルの一種であるため、通常のファイルと同様に読み書きが可能です。

### 無名と名前つき

パイプには無名と名前つきの2種類があります。名前つきパイプはファイルとしてどこかに存在し、パスで指定できます。一方、無名パイプはプログラム内で生成され、内部的に利用されます。

### 一方通行

Linuxのすべてのパイプは一方通行です。つまり、1つのパイプを利用して双方向通信を行うことはできません。双方向通信が必要な場合は、2つのパイプを使用します。

### 複数プロセスからの利用

1つのパイプは、複数のプロセスから読み取り側または書き込み側として開くことができます。ただし、パイプはバイト列として扱われるため、複数のプロセスが同時に読み書きする場合は、どのプロセスがどのデータを読むかを意識して設計する必要があります。この本では、まず1つの書き手と1つの読み手で使う例を扱います。

### 読み書きの順序

パイプでは、最初に書き込まれたデータが最初に読み出されます。

### メモリ上に存在

無名・名前つきのどちらのパイプも、読み書きに伴って記憶装置に何かが書き込まれることはありません。すべてのデータはOS内部のメモリ上に保存されます。

### 練習

ディレクトリを作成し，カレントディレクトリをそのディレクトリに変更します．

```bash
$ mkdir pipe
$ cd pipe
```

次にパイプを作成します．名前つきパイプは以下のコマンドで作成できます．

```bash
$ mkfifo a
```

ターミナルをもう一つ開きます．どちらか1つのターミナルで次のコマンドを入力します．これはパイプに文字列を書き込むコマンドです．

```bash
$ echo ABC > a
```

このコマンドはすぐには終了せず，実行中のままになります． もう一つのターミナルで，次のコマンドを入力します．

```bash
$ cat a
```

すると，このコマンドの実行は即座に終了し，後者のコマンドを入力したターミナルで，以下のように出力されるでしょう．このコマンドが終了すると，初めに入力したコマンドも終了します．

```text
ABC
```

この出力が表示されると、最初に入力したコマンドも終了します。これは、2つのターミナルのそれぞれのプロセス間で通信が成功したことを示しています。最初に入力した**`echo`**コマンドがすぐに終了しなかった理由は、パイプの読み取りを行うプロセスが存在しなかったため、書き込みが完了しなかったからです。
